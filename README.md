# Конфигурация Neovim

Современная конфигурация Neovim, оптимизированная для разработки с упором на производительность, удобство использования и эстетику.

## Обзор

Эта конфигурация Neovim построена с использованием [Lazy.nvim](https://github.com/folke/lazy.nvim) в качестве менеджера плагинов и включает тщательно подобранный набор плагинов для улучшенного опыта программирования. Конфигурация включает в себя:

- **Красивый интерфейс**: Тема Catppuccin с настраиваемой строкой состояния и вкладками буферов
- **Интеллектуальное автодополнение кода**: nvim-cmp с интеграцией LSP
- **Подсветка синтаксиса**: Tree-sitter для улучшенной подсветки синтаксиса
- **Навигация по файлам**: Проводник nvim-tree и нечеткий поиск Telescope
- **Форматирование кода**: Conform.nvim для автоматического форматирования кода при сохранении
- **Поддержка LSP**: Mason для простого управления языковыми серверами
- **Инструменты продуктивности**: Which-key для обнаружения горячих клавиш, Todo-comments для отслеживания задач

## Возможности

### Основные возможности
- **Современный интерфейс**: Тема Catppuccin со строкой состояния lualine и управлением вкладками bufferline
- **Расширенное редактирование**: nvim-autopairs для автоматического завершения скобок, nvim-cmp для интеллектуального автодополнения кода
- **Синтаксическая интеллектуальность**: Tree-sitter для продвинутой подсветки синтаксиса и понимания кода
- **Навигация**: Проводник nvim-tree, нечеткий поиск Telescope для файлов/буферов
- **Качество кода**: Conform.nvim для автоматического форматирования кода, Todo-comments для управления задачами
- **Опыт разработчика**: Noice.nvim для улучшенных уведомлений интерфейса, Snacks.nvim для вспомогательных функций

### Поддержка языков
- Полная поддержка LSP через Mason для простой установки языковых серверов
- Подсветка синтаксиса для: bash, css, html, javascript, json, lua, markdown, python, sql, toml, yaml и других
- Форматирование кода для Lua (stylua) и Python (ruff)

## Установка

1. **Предварительные требования**:
   - Neovim 0.8+
   - Git
   - Node.js (для некоторых языковых серверов)
   - Python 3 (для некоторых языковых серверов)

2. **Клонирование конфигурации**:
   ```bash
   # Сделайте резервную копию текущей конфигурации nvim
   mv ~/.config/nvim ~/.config/nvim.backup
   
   # Клонируйте эту конфигурацию
   git clone <адрес-репозитория> ~/.config/nvim
   ```

3. **Установка зависимостей**:
   ```bash
   # Установка языковых серверов (пример)
   npm install -g typescript typescript-language-server
   pip install python-lsp-server
   
   # Для форматировщиков кода
   pip install stylua ruff  # или установка через менеджер пакетов
   ```

4. **Запуск Neovim**:
   ```bash
   nvim
   ```
   При первом запуске Neovim Lazy.nvim автоматически установит все плагины.

## Использование

### Базовая навигация
- `<Ctrl-n>`: Переключить проводник файлов (nvim-tree)
- `<Leader>ff`: Поиск файлов (Telescope)
- `<Leader>fg`: Поиск по содержимому (Telescope)
- `<Leader>fb`: Поиск буферов (Telescope)

### Редактирование кода
- `<Ctrl-Space>`: Запустить автодополнение (nvim-cmp)
- `<Enter>`: Подтвердить автодополнение (nvim-cmp)
- Автоматическое форматирование при сохранении (Conform.nvim)

### Обнаружение горячих клавиш
- `<Leader>?`: Показать доступные горячие клавиши (Which-key)

### Цветовая схема
- Конфигурация использует тему Catppuccin Macchiato по умолчанию
- Чтобы изменить темы, измените `color.lua`

## Структура конфигурации

Конфигурация организована в модульные файлы Lua в директории `lua/plugins/`:

- `color.lua`: Настройки темы и цветов
- `nvim-cmp.lua`: Конфигурация движка автодополнения
- `lualine.lua`: Настройки строки состояния
- `bufferline.lua`: Настройки вкладок буферов
- `nvim-tree.lua`: Настройки проводника файлов
- `telescope.lua`: Настройки нечеткого поиска
- `treesitter.lua`: Настройки подсветки синтаксиса
- `mason.lua`: Настройки менеджера языковых серверов
- `whichkey.lua`: Настройки помощника горячих клавиш
- `todo-comments.lua`: Настройки подсветки комментариев
- `conform.lua`: Настройки форматирования кода
- `nvim-autopairs.lua`: Настройки автоматического завершения скобок
- `noice.lua`: Улучшение уведомлений интерфейса
- `snacks.lua`: Вспомогательные функции и улучшения

## Настройка

Чтобы настроить эту конфигурацию:

1. **Изменить тему**: Отредактируйте `lua/plugins/color.lua`, чтобы переключить темы или изменить цвета
2. **Добавить плагины**: Создайте новые `.lua` файлы в `lua/plugins/`, следуя существующему шаблону
3. **Изменить горячие клавиши**: Большинство горячих клавиш можно изменить в конфигурациях соответствующих плагинов
4. **Добавить поддержку языков**: Добавьте новые языки в список `ensure_installed` в `lua/plugins/treesitter.lua` и настройте LSP в `lua/plugins/mason.lua`

## Решение проблем

### Распространенные проблемы

1. **Плагины не загружаются**: Выполните `:Lazy sync` для принудительной синхронизации плагинов
2. **Языковые серверы не работают**: Проверьте `:Mason`, чтобы увидеть установленные серверы и их статус
3. **Форматирование не работает**: Убедитесь, что форматировщики установлены (stylua для Lua, ruff для Python)

### Отладка

- `:checkhealth` - Запустить встроенный проверщик состояния Neovim
- `:Lazy` - Открыть интерфейс менеджера плагинов
- `:Mason` - Открыть интерфейс менеджера языковых серверов
- `:TSInstallInfo` - Проверить статус парсеров Tree-sitter
